# The generated views — the only things `status` writes

> `status` is read-only *with respect to truth*. It writes **only two generated views** — pure regenerable projections
> whose source lives elsewhere (`shared/spine-boundary.md`: a generated view is safe to overwrite; a maintained
> artifact is not). Load this at REGENERATE VIEWS.

## Why writing these is not mutating truth

`shared/spine-boundary.md` draws a third distinction, orthogonal to declaration/realization:

- A **maintained artifact** carries state that *originates* there (the spine; the backlog ledger's execution status).
  Overwriting it would destroy the only copy — `status` **never** touches these.
- A **generated view** is a *pure, regenerable projection* of a maintained artifact — re-deriving it loses nothing.
  `AGENTS.md` (a projection of the Constitution) and `CLAUDE.md § Current State` (a projection of the artifact chain)
  are generated views. Writing them is *the definition of* keeping a derived view current — not mutating a source.

The test `status` applies before any write: *does this file carry state that exists nowhere else?* If yes, it is a
maintained artifact — do not write it. Both views fail that test (their state is fully re-derivable from the spine +
the chain), so writing them is safe.

## View 1 — `CLAUDE.md § Current State` (the machine-readable emission)

Overwrite (or create) the `## Current State` section of the **project** `CLAUDE.md` — **derived status only**. The
framework's methodology is Layer-A (framework-level) and is **never** restated in a project file. Template:
`templates/current-state.md`. Exact shape (the grader and `06` read these fields):

```markdown
## Current State
<!-- GENERATED by /status — derived status only. Do not hand-edit; re-run /status. -->

- **Spine integrity:** PASS            <!-- or: FAIL — REQ-014: registry File `capabilities/x.md` does not resolve -->
- **Sprint:** 2 · **Phase:** quality (qa verdict: SHIP)
- **Amendments:** 0 pending · 0 deferred (2 approved · 1 auto-applied)   <!-- 06-release blocks on any pending/deferred -->
- **Open [NEEDS CLARIFICATION]:** 0
- **Next command:** `/06-release sprint 2`

*Derived by /status from docs/spec/ + the artifact chain. Not a source of truth.*
```

- **Overwrite only the `## Current State` section.** Leave the rest of the project `CLAUDE.md` untouched. If the
  **section** is absent, append just the section. If the **file** is absent, create it as the **Claude bridge**:
  line 1 `@AGENTS.md` (the officially-recommended import that makes Claude Code consume the Constitution
  projection — the shape of `00-discovery/templates/claude-bridge.md`), then the section. **Never scaffold
  methodology** into a project file, and **never edit user content**: if an existing `CLAUDE.md` lacks the line-1
  `@AGENTS.md`, or the project has no `GEMINI.md` bridge (`@./AGENTS.md`), surface a one-line **advisory** in the
  report — do not write it yourself.
- **Integrity line:** `PASS`, or `FAIL — <REQ-ID>: <what broke>` (the specific L-failure — never a bare `FAIL`).
- **Next command:** the single routed command, backticked, matching the report's `→` line exactly.

## View 2 — `AGENTS.md` (the Constitution emission — full re-emit each run)

**Full re-emit each run** (the approved scope): regenerate the **entire** `AGENTS.md` from the current spine —
idempotent (byte-identical) when already in sync, **self-healing** when it had drifted from the Constitution. This
doubles as a generated-view integrity check.

The structure is the canonical emission shape (matching `00-discovery/templates/agents-view.md` — cited for shape, not
imported: `status` emits from the **spine**, a project artifact it reads, into `AGENTS.md`). The 8-part contract:

```markdown
<!-- GENERATED FILE — do not edit by hand.
     Regenerated by skill 00 (discovery) on any Tier-2/3 amendment, and by /status.
     To change it, edit the source — docs/spec/specification.md — and regenerate. -->

# <Project Name> — Agent Instructions

> A **read-only projection of this project's spine Constitution** (`docs/spec/specification.md`). … (the boilerplate
> from the template — the per-project vs framework AGENTS.md note.)

## Constitution (the project's non-negotiables)
<!-- verbatim from docs/spec/specification.md § Constitution -->
1. <constitution item 1 — copied verbatim>
2. <…>

## Where the truth lives
<!-- the fixed boilerplate: spec/ = declaration-truth; everything else references REQ-NNN -->

## Spine summary (generated)
- **Profile:** <webapp | agent-system | mcp-server | skill-pack>  <!-- mirrored from specification.md's `- **Profile:**` line; absent ⇒ webapp -->
- **Domains:** <n> — <domain list, from capabilities/*.md filenames>
- **Requirements:** <total> (`MUST` <n> · `SHOULD` <n> · `MAY` <n>) — <n> stated / <n> derived
- **Open `[NEEDS CLARIFICATION]`:** <count>  <!-- must reach 0 before release — 06-release blocks otherwise -->

_Authoritative source: docs/spec/specification.md. On any product or scope question, the Constitution wins._
```

- **Profile** — mirror the spine's `- **Profile:**` field verbatim (absent ⇒ `webapp`). It is the S7 mirror the
  downstream harness reads to know the project's shape; `status`'s router (below) also reads it to resolve the design
  phase per profile.

**How to re-emit faithfully:**
- **Project Name** — from the spine index title (`# Specification — <Name>` → `<Name>`).
- **Constitution** — copy the numbered items from `specification.md § Constitution` **verbatim** (this is the faithful
  emission the eval checks; do not paraphrase).
- **Spine summary** — recompute from the registry: domain count = distinct `capabilities/*.md` files; the RFC-2119
  priority tallies and the stated/derived split from the registry `Priority`/`Status` columns; the marker count = the
  same `n_markers` from GOVERN.
- Keep the **"do not edit" header** and the boilerplate sections intact.

**The self-healing property:** if someone hand-edited the spine Constitution and forgot to regenerate, the old
`AGENTS.md` is stale; `status` re-emitting brings the view back into sync. If nothing drifted, the re-emit is
byte-identical (a no-op diff) — which is exactly why a full re-emit is safe for a read-only-to-truth skill.

## The honesty proxy (what the eval asserts)

After `/status`, `git diff --name-only` shows **only** `CLAUDE.md` and `AGENTS.md` changed. `docs/spec/**` and every
realization are **byte-identical**. If any spine or realization file shows in the diff, `status` did something it must
not — stop and report it, don't ship the change.
