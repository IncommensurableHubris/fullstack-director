# ADR-001: Node std-lib HTTP + PostgreSQL on an EU host

- **Status:** Accepted
- **Satisfies:** the architecture-constraints envelope (Node std-lib · PostgreSQL EU · 2+ instances).

## Context
The digest must be served to multiple stateless instances sharing one store, in the EU.

## Decision Outcome
Node.js standard-library HTTP server; **PostgreSQL** (client-server, EU-hosted) as the shared datastore.

## Consequences
- **Rule:** the datastore is a client-server DB. **Prevents** an embedded/in-memory store (incompatible with 2+ instances).

## Consequences (Rule)
- **Binds** the datastore choice; **Prevents** SQLite / in-memory.
