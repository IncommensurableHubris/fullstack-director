# ADR-001: Plain Node.js + built-in `node:test`, zero runtime dependencies

- **Status:** Accepted
- **Satisfies:** the `architecture-constraints` zero-dependency mandate; Q-01 (offline, deterministic tests)

## Context and Problem Statement

The digest slice is pure domain logic (assemble + render a daily digest). It must be buildable and verifiable with no
network access and no install step, so the whole slice runs anywhere Node runs.

## Decision Outcome

Implement the digest core in **plain Node.js** (no framework, no transpile) and test it with the **built-in
`node:test`** runner plus `node:assert/strict`. **No runtime or dev dependencies** are added.

- **Binds:** all source under `src/**`; all tests use `node:test` and run via `node --test`.
- **Prevents:** adding any npm dependency; introducing a build/transpile step; reading the wall clock or randomness
  inside the core.
- **Rule (Confirmation):** `node --test` exits 0 with a positive test count, offline, on two consecutive runs; the
  dependency manifest lists **zero** runtime dependencies.

## Consequences

- Good: nothing to install; deterministic, offline verification; the reviewer re-runs the exact same command.
- Bad / accepted: no assertion-library sugar (mitigated by `node:assert/strict`); UI/web verification waits for the
  container sprint.
