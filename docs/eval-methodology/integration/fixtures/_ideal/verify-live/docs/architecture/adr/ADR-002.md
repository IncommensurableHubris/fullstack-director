# ADR-002: Adopt OpenClaw for the agent loop

- **Status:** Accepted
- **Category:** model-binding
- **Satisfies:** REQ-001 · architecture-constraints "Framework(s)"
- **Supersedes:** none
- **Resolves amendment:** n/a
- **Verified-against:** docs/verification/openclaw.md (openclaw@0.4.2)

## Context & Problem Statement

The Beacon agent must run on OpenClaw (a stated constraint). OpenClaw is too new for reliable recall, so its loop
API was verified against live docs/source before this decision (see the Verified-against record).

## Decision Outcome

**Chosen:** OpenClaw, because it is the mandated host framework. The agent loop is entered through `Claw.run(task)`
(verified against `docs/verification/openclaw.md`, openclaw@0.4.2), and tools register via `claw.tool(...)`.

- **Binds:** the agent loop to OpenClaw's `Claw.run` entry point.
- **Prevents:** hand-rolling a loop outside the framework.
- **Rule (Confirmation):** `src/agent.ts` imports `Claw` from `openclaw` and calls `Claw.run`.

## Consequences

- **Positive:** the framework handles retries + tracing.
- **Negative:** the build is bound to OpenClaw's release cadence — a version bump re-verifies the record.
