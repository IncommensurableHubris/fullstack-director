# Fullstack Director — Claude Code bridge

[`AGENTS.md`](AGENTS.md) (imported above) is the portable entry point and applies on every harness. This file adds
the **methodology + governance** layer and the **Claude-specific** operating notes. It is the *index* to the
methodology, not a restatement of it.

## Methodology

Fullstack Director is **reuse-first and gate-driven**: each skill keeps its proven machinery — assumption-mapping
challenge, explicit gates, verification states, the DM-ID manifest — and changes only its reads/writes to the spine
and its Reconcile wiring. The craft behind each phase lives in that skill's own `references/`; the *why* lives in
the design-record commits (git history); the spine model and the amendment/subagent protocols live in
[`shared/`](shared/). The framework's **own intent** — mission + non-negotiables — lives in
[`docs/charter.md`](docs/charter.md) (the Layer-A declaration layer): doctrine changes cite the charter item they
serve, and a change contradicting one is **user-gated, always**.

## Governance precedence

Two sources of truth, two derived views. When the framework runs **on a project**, authority resolves so:

1. **The project's spine Constitution** (`docs/spec/specification.md`) — *this project's intent.* It **wins on any
   product or scope question.** It holds only that project's non-negotiables and **must not restate framework
   principles** (those are methodology — they live here, in `shared/`, and in the skills' references).
2. **Framework methodology** (this file + `shared/` + the skills' references) — *how* the work gets done.
   Subordinate to the project's intent on product/scope; authoritative on process.
3. **The project's `AGENTS.md`** — a **read-only emission of the Constitution**, regenerated by `00-discovery` on
   any Tier-2/3 amendment and by `status`. No independent authority: edit the Constitution and regenerate, never
   hand-edit the emission.
4. **Project `CLAUDE.md`** — two framework-owned generated pieces: **line 1 `@AGENTS.md`** (the Claude Code bridge
   to the Constitution emission — created by `00-discovery` only when the file is absent; an existing user file is
   advised, never edited) and **§ Current State** (**derived status only**, written by `status`). Never a source
   of truth; the rest of the file is the user's. (`GEMINI.md` = the same bridge for Gemini CLI, one line
   `@./AGENTS.md`; Codex reads `AGENTS.md` natively.)

> Items 3–4 are *project* files (Layer B) — don't confuse them with **this framework repo's** `AGENTS.md`/`CLAUDE.md`
> (Layer A). Same filenames, different repos, no collision.

## Running under Claude Code

- **Subagents are available.** The capability tier in [`shared/subagent-protocol.md`](shared/subagent-protocol.md)
  takes its *spawn* branch here — skills `05-reviewer`, `03-architect` (reconciler), and `07-security` (panel) spawn
  real subagents via the Task/Agent tool. Isolation is real **only** when the spawner is a fresh session.
- **Skill discovery.** Claude Code auto-discovers `.claude/skills/`; the canonical skills live in `.agents/skills/`.
  A Claude dev bridge (a `.claude/skills` link, a small sync, or `--plugin-dir`) is **deferred** — resolve it when
  first needed. The eval harness loads skills by explicit path regardless.
